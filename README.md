# 서비스 시나리오
- 기능적 요구사항
    + 고객이 목적지를 설정하고 택시배차를 요청한다.
    + 고객이 결제한다.
    + 결제가 완료되면 요청내용(승차장소, 목적지)이 택시기사에게 전달된다.
    + 택시기사는 수신된 배차정보를 승인하고 승차장소로 이동한다.
    + 고객이 배차요청을 취소할 수 있다.
    + 배차요청이 취소(결제취소)가 되면 배차도 취소된다.
    + 택시기사는 배차요청/승인 상태를 중간중간 조회한다.
    + 배차상태가 바뀌면 카톡을 알림을 보낸다.
- 비기능적 요구사항
    + 트랜잭션
    + 장애격리
